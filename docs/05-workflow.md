# Workflow

## Applicant Workflow
1. Visit login or register page.
2. Create account or sign in.
3. Verify email if required.
4. Open enrollment dashboard.
5. Start or continue enrollment form.
6. Complete student, parent, medical, agreement, and document steps.
7. Submit enrollment application.
8. Submit enrollment fee payment proof.
9. Wait for admissions and finance review.
10. Receive approval or rejection status.

## Draft Workflow
1. Applicant starts form.
2. System saves progress locally and to backend.
3. Applicant can leave and return.
4. Dashboard shows progress percentage and last saved time.
5. Applicant continues from last step.

## Review Workflow
1. Admissions reviews applicant details and documents.
2. Rejected documents are marked with reasons or status.
3. Applicant re-uploads rejected documents.
4. Admissions approves when requirements are complete.

## Payment Workflow
1. Applicant selects payment method.
2. Applicant follows payment instructions.
3. Applicant uploads receipt or proof.
4. Finance verifies or rejects payment.
5. Dashboard shows payment status.

## Release Workflow
1. Make code changes.
2. Run frontend build.
3. Run relevant Laravel checks or manual route checks.
4. Review UI in browser.
5. Commit only related files.
6. Deploy using the project deployment checklist.
