# UI UX Design

## Design Goal
Make the enrollment flow calm, trustworthy, and easy to finish on desktop or mobile.

## Shared Components
- Guest layout
- App layout
- Enrollment header
- Dashboard cards
- Form inputs
- Select inputs
- Textarea inputs
- Buttons
- Status badges
- File upload component
- Loading screens
- Skeleton loaders

## Global CSS Files
- `resources/css/global.css`
- `resources/css/auth.css`
- `resources/css/dashboard.css`
- `resources/css/enrollment.css`
- `resources/css/app.css`

## UI Rules
- Use consistent spacing, border radius, and shadows.
- Keep primary actions visually clear.
- Use green as the main school/system color, with restrained accents.
- Avoid overcrowded cards.
- Keep form steps short and scannable.
- Use sticky actions only where they help completion.

## UX Rules
- Show progress clearly.
- Save drafts automatically.
- Give useful validation messages.
- Make locked actions understandable.
- Show payment and application statuses separately.
- Make mobile form controls large enough to tap.

## Loading UX
- Use one splash loader only on first visit.
- Keep loading animations short and purposeful.
- Use skeletons for dashboard/form content where data may take time.
